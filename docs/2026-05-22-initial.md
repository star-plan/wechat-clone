# wechat-clone 开发方案

## 目标

开发一个 macOS 微信分身管理 CLI 工具，用于自动创建、修复、删除、启动多个微信分身 App。

核心思路：

1. 复制 `/Applications/WeChat.app`
2. 修改分身 App 的 `CFBundleIdentifier`
3. 重新签名
4. 清除 quarantine 属性
5. 支持批量管理

## 技术选型

- 语言：Go
- 平台：macOS
- 架构：优先支持 Apple Silicon，兼容 Intel Mac
- 交付方式：
  - GitHub Release 二进制
  - 后续支持 Homebrew Tap

```text

```

## Bundle ID 规则

原始微信：

```text
com.tencent.xinWeChat
```

分身：

```text
com.tencent.xinWeChat.clone1
com.tencent.xinWeChat.clone2
com.tencent.xinWeChat.clone3
```

## 核心功能

### create

创建指定数量的微信分身。

示例：

```bash
wechat-clone create 3
```

流程：

1. 检查 `/Applications/WeChat.app` 是否存在
2. 检查参数是否合法
3. 循环创建分身
4. 若目标 App 已存在，提示跳过或覆盖
5. 复制 App
6. 修改 `Info.plist` 中的 `CFBundleIdentifier`
7. 执行重新签名
8. 清除 quarantine 属性
9. 输出创建结果

### list

列出当前已有分身。

```bash
wechat-clone list
```

输出示例：

```text
WeChat Clone 1.app  com.tencent.xinWeChat.clone1
WeChat Clone 2.app  com.tencent.xinWeChat.clone2
```

### open

打开所有微信分身。

```bash
wechat-clone open
```

也支持打开指定编号：

```bash
wechat-clone open 2
```

### repair

重新修复签名和 quarantine。

```bash
wechat-clone repair
wechat-clone repair 2
```

流程：

1. 检查分身是否存在
2. 重新设置 Bundle ID
3. 重新 codesign
4. 清除 quarantine

### remove

删除分身。

```bash
wechat-clone remove 2
wechat-clone remove all
```

删除前需要二次确认，除非提供 `--force`。

### doctor

检查环境。

检查项：

* 当前系统是否为 macOS
* `/Applications/WeChat.app` 是否存在
* `codesign` 是否可用
* `PlistBuddy` 是否可用
* 当前用户是否有 `/Applications` 写入权限
* 已存在的微信分身状态

## 系统命令

复制 App：

```bash
cp -R "/Applications/WeChat.app" "/Applications/WeChat Clone 1.app"
```

修改 Bundle ID：

```bash
/usr/libexec/PlistBuddy -c "Set :CFBundleIdentifier com.tencent.xinWeChat.clone1" "/Applications/WeChat Clone 1.app/Contents/Info.plist"
```

重新签名：

```bash
codesign --force --deep --sign - "/Applications/WeChat Clone 1.app"
```

清除 quarantine：

```bash
xattr -rd com.apple.quarantine "/Applications/WeChat Clone 1.app"
```

打开 App：

```bash
open "/Applications/WeChat Clone 1.app"
```

## 错误处理要求

需要清晰提示：

* 未找到微信
* 没有权限写入 `/Applications`
* 分身已存在
* 签名失败
* 修改 plist 失败
* 当前系统不是 macOS

错误信息要适合普通用户阅读，不要只输出 Go error。

## 安全约束

不要做以下事情：

* 不修改原始 `/Applications/WeChat.app`
* 不关闭 SIP
* 不注入 dylib
* 不实现防撤回
* 不绕过账号、安全、风控系统
* 不收集用户信息
* 不上传任何数据
